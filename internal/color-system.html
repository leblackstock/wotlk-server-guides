Rules at a glance

1

Class frames the page

Paladin pink owns titles, section landmarks, general links, class notes, and the dominant class-level ambience.

2

Spec signals state

Protection blue marks the current chapter, Protection-specific controls, focus or hover state, and technical emphasis.

3

Navigation is tiered

Guide Hub is neutral silver, inactive chapter links are neutral, and only the active chapter receives the spec accent.

4

Status is fixed

Success, warning, danger, and information never change when the class or specialization changes.

5

Mechanics are independent

Rotation and pull categories use named mechanic roles rather than borrowing class, spec, status, or item-quality meaning.

6

Color supports labels

Text, icons, or component shape must carry the meaning. Color is never the only instruction.

Dark-mode foundations

Semantic status colors

These meanings are global. A Protection information note stays cyan; a Holy warning stays amber; a Retribution danger state stays danger red.

Success / safe

Confirmed requirements, completed checks, and safe states.

Warning

Caution, a meaningful tradeoff, or something that needs verification.

Danger

Failure risk, emergency response, or a harmful correction.

Information

Neutral server notes, supporting explanation, and reference information.

Mechanic color boundaries

The reusable 969 pair is formalized in the shared token sheet because it appears on multiple Tankadin pages. Pull-category aliases remain local to the Protection stylesheet. Neither layer borrows class, spec, status, or item-quality names for its meaning.

969 rotation groups

9-second group 6-second group

9-second: violet #B59CFF, soft #DED5FF. 6-second: cyan #72E3FF, soft #BCEFFF. Visible labels and spell icons remain mandatory.

Protection pull categories

Trash Pickup Boss Raid Utility Taunt

Trash and taunt use cyan; pickup uses amber; boss uses Paladin pink; raid uses violet; utility uses green. These are local category mappings, and the labels carry the meaning.

Implementation reference

Activate a guide theme

Load the token sheet before component styles and identify the guide on its body element.

<link rel="stylesheet" href="../assets/guide-color-system.css">
<link rel="stylesheet" href="../assets/style.css">
<body data-guide-class="paladin" data-guide-spec="protection">

Consume role aliases, not raw colors

Class identity, active spec state, and global navigation are separate roles even when they appear in the same component row.

.page-title { color: var(--class-accent); }
.site-nav a[aria-current] { border-color: var(--theme-accent); }
.guide-hub-link { color: var(--navigation-global-soft); }

Central token ownership

  • Foundation surfaces, text, borders, and global navigation
  • Class and specialization identities
  • Fixed semantic status meanings
  • Item quality and socket colors
  • Approved reusable mechanic pairs such as 969

Guide-specific ownership

  • Local aliases that consume approved mechanic tokens
  • Pull categories and encounter-category mappings
  • Auction House market and demand categories
  • One-off data visualizations

Site-section identities

A site section may have a restrained identity without borrowing class, specialization, semantic-status, or item-quality meaning. The Addon Library is the first formal example.

Addon catalog identity

--section-addons-accent: #8fa8f5, --section-addons-soft: #d4deff, and --section-addons-deep: #2d3c6b. The base accent has approximately 6.61:1 contrast against #1e2630.

1

Identity boundaries

Section identity is separate from class/spec identity. Small class metadata accents are allowed, but class colors must not recolor the whole filter system.

2

Independent meanings

Recommendation is separate from semantic status. Compatibility is separate from recommendation. An essential addon can still carry an amber version warning.

3

Neutral professions

Profession tags remain neutral until a formal profession identity system exists. Do not invent a rainbow of profession or filter colors.

Filter states

Unselected Selected ✓ Disabled

Selected filters use the section accent, while unselected filters stay neutral. Disabled choices reduce opacity but retain readable labels. Meaning never relies on color alone.

Recommendation, purpose, and compatibility

★ Essential · Core Function ✓ Recommended · Utility ○ Optional · Training ◎ ! Old / verify version

Importance uses icons and text. Training and Aesthetic are neutral purposes, not warnings. Old or uncertain compatibility uses the semantic warning color. YouTube brand red is permitted only as a limited external-brand mark, never as a site status color.

Do not create rainbow filters. The Addon Library uses one section accent, neutral metadata chips, semantic warning colors only when warranted, and written labels for every meaning.
